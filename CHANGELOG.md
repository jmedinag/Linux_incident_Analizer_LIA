# Changelog

All notable changes to this project will be documented in this file.

The format is inspired by [Keep a Changelog](https://keepachangelog.com/en/1.1.0/)
and this project follows [Semantic Versioning](https://semver.org/).

---

## [1.0.0] - 2026-07-13

### Added

- Initial public release of Linux Incident Analyzer (LIA).
- Modular architecture for Linux incident analysis.
- CPU analysis module.
- Memory analysis module.
- Disk and filesystem analysis module.
- Network diagnostics module.
- Services analysis module.
- System log analysis module.
- System information collection.
- Structured incident report generation.
- Modular recommendation engine.
- Colorized terminal output.
- Installation script (`install.sh`).
- Uninstallation script (`uninstall.sh`).
- Project documentation.
- MIT License.

### Changed

- Initial project structure organized into modules and reusable libraries.

### Known Limitations

- Recommendation engine currently provides static guidance.
- Historical incident correlation is not yet available.
- Knowledge-based recommendations are planned for future releases.
- AI-assisted incident investigation is planned for future releases.
